Importing a CoreData model

Although you could create a schema directly on the PostgreSQL server the BaseTen Assistant makes it easy to import a CoreData model designed in XCode, and use it to create a database schema.

To import a CoreData model:

Connect to your database server

  • Enter the database server’s address and port number, or pick a PostgreSQL server using the Bonjour pop-up menu.
  • Enter the database name and your user account details.
See “Connecting to PostgreSQL” if you have problems connecting.

Choose File → Import Data Model… ⌘O, and select an .xcdatamodel file.

Choose which entities you want to import.

In case your data model has different configurations, you have the option to select the one from which you'd like to import the entities.

If you want the tables to be placed in some other schema than public, enter its name in the Schema Name field.

Clicking Dry Run will output the SQL statements in the Log window before actually sending them to the database.

Install the BaseTen schema.

If the database does not have the BaseTen supporting schema installed the assistant will offer to install it. The schema provides support needed for Postgres to notify the framework of changes to data in a similar manner to Key/Value Observing.

Review model compiler errors.

BaseTen Assitant will inform you of any errors it found while compiling your model file. You can cancel the operation to resolve the errors manually, or press Continue and allow the assistant to attempt to resolve them automatically.

See “Resolving model compiler errors”

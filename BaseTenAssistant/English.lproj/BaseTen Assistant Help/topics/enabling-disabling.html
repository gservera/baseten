Enabling and disabling entities

BaseTen updates its objects based on database notifications it receives. To receive notifications from tables and views, they need to be BaseTen-enabled. In practice, some triggers and rules are added into them and tables are created into the BaseTen schema.

  • Enabling and disabling tables is done using the checkboxes in the Assistant's main window. A primary key is needed in each table.
  • Views also need a primary key. A primary key for use with BaseTen can be specified with the Assistant.

To enable a view:

Click “Setup…” next to the view that you want to enable. This opens the inspector.

Select the primary key columns.

Finally check “BaseTen enabled.”
